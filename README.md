## Робота програми

На зображенні нижче можна бачити головний екран розробленого застосунку, який містить текстове поле, куди користувач вводить інформацію. На екрані також присутні кнопки, через які користувач взаємодіє із програмою. При натисненні кнопки "ОК" виводиться введений користувачем текст на екран обраним кольором. При натисненні кнопки "Cancel" напис прибирається, а ввід користувача очищується.

![Головний екран. Лабораторна робота 1](/images/lab1/1.png)

При спробі натиснути кнопку "ОК" користувач отримує сповіщення про необхідність заповнення всіх ввідних даних. Якщо користувач заповнює всі необхідні дані, то лише тоді запрацює передбачений кнопкою "ОК" сценарій.

![Валідація вводу. Лабораторна робота 1](/images/lab1/2.png)

Якщо заповнено лише одна з форм заповнення вхідних даних, то програма все одно вимагатиме від користувача ввести всі дані.

![Неповний ввід. Лабораторна робота 1](/images/lab1/3.png)

Після обрання потрібного кольору та вводу тексту, натискаємо на кнопку "ОК". Тоді користувач побачить напис потрібним кольором.

![Коректна робота програми. Лабораторна робота 1](/images/lab1/4.png)

При натисненні кнопки "Cancel" ввід користувача та написи очищуються, як показано на зображенні нижче.

![Робота кнопки Cancel. Лабораторна робота 1](/images/lab1/5.png)

# Лабораторна робота 2

## Основні відомості

**Умова завдання відповідно до варіанту**: *Написати програму під платформу Андроїд, яка має інтерфейс, побудований здекількох фрагментів згідно варіанту. Перший фрагмент представляє з себе форму для введення даних та кнопку підтвердження («OK»), а інший фрагмент відображає результат взаємодії. Тобто другий фрагмент містить тестове поле з результатом та кнопкою «Cancel» (якщо згідно варіанту така існує, якщо ж за варіантом її немає – можете додати за власним бажанням), яка очищає або приховує (або видаляє) другий фрагмент та очищає форму введення з першого фрагменту. Зверніть увагу, що робота з фрагментами відбувається в рамках однієї Діяльності.*

Примітка: завдання відповідає варіанту лабораторної роботи No1.

# Лабораторна робота 3

**Умова до лабораторної роботи відповідно до варіанту**: *Написати програму під платформу Андроїд, яка доповнює програму, що розроблена за лабораторною роботою 2, роботою зі сховищами. Тобто при натисканні на кнопку «ОК» додатково:

Якщо дані відсутні (сховище пусте) відобразити відповідне повідомлення. За бажанням можна додатково реалізувати оновлення та видалення даних зі сховища.*

## Реалізація сховища у лабораторній роботі

Робота зі сховищем у лабораторній роботі №3 була реалізована за допомогою створення нового класу, який відповідає за роботу із базою даних. Цей клас має назву `DBHelper`, який успадковується від бібліотечного класу `SQLiteOpenHelper`. Клас містить основні методи взаємодії із базою даних, яка є внутрішнім сховищем додатку.

![Клас для роботи зі сховищем. Лабораторна робота 3](/images/lab3/1.png)

В якості даних, що відправляються до бази даних використовуватимемо текст написаний користувачем, колір обраний ним та момент часу виконання такого запиту після натиснення кнопки "ОК". Для реалізації такої особливості було реалізовано клас `Request`, який містить відповідні поля, включно з унікальним ідентифікатором, який є первинним ключем такої сутності, що представлений полем `public int id`.

![Клас запису. Лабораторна робота 3](/images/lab3/request.png)

## Робота програми

Головний екран сторінки являє собою аналогічний до того, який був розроблений у [лабораторній роботі №2](#лабораторна-робота-2). Він містить основні поля вводу, результат програми. Відмінність полягає у кнопці "Open", яка відкриває нову активність із переліком записів у базі даних.

**Нова активність `Second Activity`**

Для роботи із вмістом бази даних створено нову активність, яка представлена класом `Second Activity`, що успадковує бібліотечний клас `AppCompatActivity`. 

![Клас нової активності](/images/lab3/second_activity.png)

Після виконання запиту програми можемо бачити її результат на головному екрані. Виведено напис обраним користувачем кольором.

![Робота програми](/images/lab3/3.png)

Натиснемо кнопку "Open" та переглянемо результат збереження записів до бази даних. При натисненні відкривається нова активність, яка містить перелік виконаних запитів до бази даних. Можемо бачити кнопку "Back", яка забезпечує перехід до головної активності.

## Робота програми

При відкритті головного вікна можемо бачити наступний головний екран, який містить дві кнопки:
- "Завантажити": відповідає за викачування в додаток аудіо- або відеозапису із сховища девайсу.
- "Викачати": відповідає за викачування в додаток аудіо- або відеозапису із інтернету із використанням URL-адреси

![Сховище пристрою](/images/lab4/2.png)

Після обрання відповідного файлу можемо бачити його у переліку доступних для взаємодії.

![Аудіофайл у списку](/images/lab4/3.png)

![Програвання аудіо](/images/lab4/4.png)

Можемо викачати файл із інтернету вставивши посилання на нього.

![Викачування файлу](/images/lab4/6.png)

Після викачування файл доступний у переліку і ми також можемо запустити його на програвання
